# Plan 001: Project Initialization

**Date:** 2026-07-04
**Status:** superseded
**Superseded by:** 002-2026-07-04-crm-core-refactor.md

## Summary

Bootstrapped the full-stack Office Chore Management App as defined in CLAUDE.md.

## What Was Done

### Phase 1: Project Scaffolding
- Created directory structure for backend and frontend
- Initialized backend with Express, Prisma, TypeScript, and all dependencies
- Initialized frontend with Vite, React 18, TanStack Query, FullCalendar, Tailwind CSS v4
- Configured Vite proxy to backend port 4000

### Phase 2: Database Schema
- Defined Prisma schema with User, Chore, Notification models
- SQLite used with string fields for enums (SQLite limitation)
- Ran initial migration

### Phase 3: Backend Implementation
- Express app with CORS, JSON parsing, route mounting
- JWT authentication middleware with role-based guards
- Full CRUD for chores with recurrence support
- User management (admin only)
- Reports endpoints (stats, overdue, history)
- Notification system with polling support
- Email service (optional Gmail SMTP)
- Daily cron job for overdue detection
- Admin seeding on first run

### Phase 4: Frontend Implementation
- Axios client with auth interceptor
- Auth context with localStorage persistence
- Protected and admin-only routes
- Login page with registration toggle
- Chores page with status filters and CRUD
- Calendar page with FullCalendar integration
- Reports page with stats cards and history
- Team management page (admin only)
- Notification bell with 30s polling
- Consistent slate/indigo design with Tailwind

### Phase 5: Governance
- This plan file created

## Decisions
- Used SQLite string fields instead of enums (SQLite doesn't support enums)
- Used Tailwind CSS v4 with @tailwindcss/vite plugin
- Used PrismaClient singleton pattern
- Express v5 (handles async errors natively)
- React 18 (per spec, despite React 19 being latest)
