# מדריך פעולות בעלים — כל מה שנשאר לעשות, מא' עד ת'

עדכון אחרון: 2026-07-10

זה המדריך המלא של **הפעולות שרק אתה יכול לעשות** (חשבונות, תשלומים, אישורים חיצוניים).
כל מה שקוד — כבר בנוי ובדוק. כל שלב כאן מסומן מי מבצע: 👤 אתה | 🤖 עמית+Claude ביחד.

**סדר הביצוע המומלץ: מסלול 1 → 2 → 3 → 4 → 5. מסלול 6 אופציונלי בהמשך.**

## מסלול 1 — מספר טלפון ייעודי לוואטסאפ 👤

**למה:** רישום מספר ל-WhatsApp Cloud API מנתק אותו מאפליקציית וואטסאפ הרגילה — לכן אסור להשתמש במספר האישי או במספר עסקי שמשמש לצ'אט. הוחלט (תוכנית 010): מספר חדש ייעודי.

**דרישות מהמספר:**
- מסוגל לקבל SMS או שיחה קולית **פעם אחת** (לאימות OTP ברישום)
- לא רשום לוואטסאפ (רגיל או Business App) — אם היה פעם, צריך למחוק את החשבון קודם
- נשאר בבעלותך לתמיד (אם המספר יתנתק/יוחזר לספק — תאבד את הזהות מול הלקוחות)

**אפשרויות (לפי המלצה):**
1. **eSIM/SIM ישראלי זול** — קו נוסף אצל הספק שלך או ספק זול (019, רמי לוי וכו'). עלות: ~10-20 ₪/חודש. הכי פשוט ואמין.
2. **מספר וירטואלי ישראלי** מספק VoIP שתומך בקבלת SMS. זול יותר אבל ודא שהוא מקבל SMS אמיתי (רבים לא).

**מה לעשות:**
1. רכוש את הקו והכנס את ה-SIM לטלפון כלשהו (רק לרגע ה-OTP)
2. ודא שהוא מקבל SMS (שלח לעצמך הודעת בדיקה)
3. **אל תתקין עליו וואטסאפ!**
4. שמור את המספר — נשתמש בו במסלול 3

✅ **סיום המסלול:** יש לך מספר שמקבל SMS ולא מחובר לוואטסאפ.

## מסלול 2 — דיפלוי ל-Railway 👤 + 🤖

**למה:** בלי כתובת HTTPS ציבורית — Make לא יכול לשלוח לידים, טופס האתר לא מתחבר, ומטא לא יכולה להחזיר הודעות נכנסות. זה השער לכל השאר. (תוכנית 010 Phase F — מפורטת שם.)

**עלות:** Railway Hobby — ‎$5/חודש (כולל את המשאבים שהמערכת צריכה; SQLite על דיסק קבוע).

### שלב א — פתיחת חשבון 👤
1. גש ל-https://railway.com → **Login** → התחבר עם חשבון ה-GitHub שלך (`amitoa50`) — זה גם נותן ל-Railway גישה לריפו
2. שדרג ל-**Hobby plan** (Settings → Plans) והזן אמצעי תשלום

### שלב ב — הכנות שאתה מחליט 👤
בחר ושמור במקום בטוח (מנהל סיסמאות):
- **אימייל + סיסמה לאדמין הראשון בייצור** (`ADMIN_EMAIL` / `ADMIN_PASSWORD`) — לא admin123!
- החלטה: האם גם צוות נוסף מקבל משתמשים ביום הראשון

### שלב ג — הקמת השירות 👤 + 🤖 (ביחד, אני מכווין צעד-צעד)
1. New Project → **Deploy from GitHub repo** → בחר `amitoa50/innokids-crm`
2. נגדיר יחד: פקודות build/start, **Volume** קבוע ל-`/data` (הדאטהבייס), ומשתני הסביבה:
   - `NODE_ENV=production`, `DATABASE_URL=file:/data/prod.db`
   - `JWT_SECRET` (אני אייצר חזק), `WEBHOOK_API_KEY` (אני אייצר חזק — נצטרך אותו שוב ב-Make!)
   - `ADMIN_EMAIL` / `ADMIN_PASSWORD` (שבחרת בשלב ב)
   - `WHATSAPP_PROVIDER=mock`, `AUTOMATION_ENABLED=false` — עד מסלול 3
3. Deploy → נוודא `GET /api/health` עובד ותתחבר עם האדמין החדש
4. נגדיר גיבוי יומי ל-SQLite

✅ **סיום המסלול:** כתובת ציבורית `https://<something>.up.railway.app`, אתה מחובר, המערכת חיה 24/7.

## מסלול 3 — וואטסאפ אמיתי (אחרי מסלולים 1+2) 👤 + 🤖

**מה כבר יש:** אפליקציית מטא (Innokids-Bot), System User עם טוקן קבוע, App Secret, וכל הקוד. חשבונות הטסט נזנחו בכוונה — מתחילים נקי עם המספר האמיתי. (הצ'קליסט המלא: `META_WHATSAPP_SETUP.md` + תוכנית 010 Phases A-E.)

### שלב א — WABA אמיתי + רישום המספר 👤 (אני מלווה מסך-מסך)
1. business.facebook.com → הגדרות עסק → **חשבונות → חשבונות וואטסאפ** → הוסף → צור WABA חדש
2. developers.facebook.com → Innokids-Bot → WhatsApp → קשר את ה-WABA החדש
3. הוסף את **המספר ממסלול 1**: הזן, קבל OTP ב-SMS, אשר
4. הגדר שם תצוגה **INNOKIDS** (ייתכן תהליך אישור קצר של מטא לשם)
5. שמור: **Phone Number ID** ו-**WABA ID** החדשים (תעביר לי — אני מעדכן בסביבת הייצור)
6. **אמצעי תשלום:** WhatsApp Manager → הגדרות → תשלומים → הוסף כרטיס. בלעדיו תבניות לא נשלחות. (שיחות שירות — חינם; שיחות תבנית Utility/Marketing — אגורות בודדות לשיחה, לפי תעריף מטא לישראל)
7. ודא שלטוקן של `Innokidscrmapi` יש גישה ל-WABA החדש (הגדרות עסק → System Users → נכסים) — אחרת ניצור טוקן חדש (2 דק')

### שלב ב — תבניות 🤖 (אני, באישורך)
1. אני מגיש את 12 התבניות ל-WABA החדש דרך ה-API (כמו שעשינו בטסט — 2 דקות)
2. 👤 אתה עוקב אחרי אישורי מטא ב-WhatsApp Manager → Message templates (בדרך כלל דקות-שעות, עד 48 שעות)
3. ⚠️ אם מטא מסווגת תבניות של סולם הניסיון כ-MARKETING (קרה בטסט) — מגישים **ערעור קטגוריה** מאותו מסך; תזכורת לפגישה היא UTILITY קלאסית
4. כשמאושר — 👤 אתה מסמן ב-CRM (מסך אוטומציה → סטטוס תבנית) או שאני מסנכרן

### שלב ג — Webhook + הפעלה מדורגת 🤖 (על שרת הייצור — קל)
1. אני מגדיר את ה-webhook על הכתובת הציבורית (verify token כבר קיים)
2. 👤 בדיקה ראשונה: שלח הודעת וואטסאפ מהטלפון שלך למספר החדש → נוודא שנוצר ליד במערכת
3. אני מדליק אוטומציות אחת-אחת (תוכנית 010 Phase E) על ליד בדיקה — אתה מקבל את ההודעות לטלפון ומאשר שהכול נכון
4. מדליקים הכול → **וואטסאפ חי** 🎉

✅ **סיום המסלול:** הורה כותב למספר — נוצר ליד; המערכת שולחת אוטומציות אמיתיות לטלפונים אמיתיים.

## מסלול 4 — Make: מודעות פייסבוק/אינסטגרם → CRM (אחרי מסלול 2) 👤 + 🤖

**מה כבר יש:** ה-webhook בצד המערכת בנוי, בדוק ועם דדופ כפול. אפס קוד נדרש. (תוכנית 011.)

**דרישות מוקדמות 👤:**
- עמוד פייסבוק של INNOKIDS עם **טופסי לידים** (Lead Ads) — קיימים או שתיצור בקמפיין
- אתה **אדמין של העמוד** בחשבון הפייסבוק האישי שלך

**עלות:** Make חינם עד 1,000 פעולות/חודש (~500 לידים). מעבר לזה — Core ‎~$9/חודש.

### שלבים 👤 (אני מכווין בכל מסך):
1. פתח חשבון ב-https://make.com (חינם)
2. Create new scenario → מודול ראשון: **Facebook Lead Ads → Watch Leads** (טריגר מיידי) → Connect → התחבר עם פייסבוק → בחר את עמוד INNOKIDS → **All forms**
3. מודול שני: **HTTP → Make a request**:
   - URL: `https://<הכתובת מ-Railway>/api/lead-intake/webhook/facebook`
   - Method: POST | Header: `x-api-key: <ה-WEBHOOK_API_KEY ממסלול 2>`
   - Body (JSON): מיפוי השדות — `full_name`, `phone_number`, `email`, `campaign_name` (=שם הטופס), `leadgen_id` (=Lead ID)
4. הפעל את הסנריו (ON, scheduling: Immediately)
5. **ולידציה:** developers.facebook.com/tools/lead-ads-testing → שלח ליד בדיקה → ודא: ירוק ב-Make, הליד מופיע ב-CRM עם המקור והקמפיין, שליחה חוזרת לא יוצרת כפול
6. הגדר התראות מייל על שגיאות ב-Make (Settings → Notifications)

✅ **סיום המסלול:** הורה ממלא טופס במודעה → מופיע ב-CRM תוך שניות → (עם מסלול 3) מקבל וואטסאפ אוטומטי תוך דקות.

## מסלול 5 — טופס האתר → CRM (אחרי מסלול 2) 👤

**מה כבר יש:** endpoint מוכן: `POST https://<הכתובת>/api/lead-intake/webhook/website` עם header `x-api-key`.

**מה צריך ממך:** תלוי איך האתר שלך בנוי —
- **אם יש לך גישה לקוד/מפתח אתר:** להעביר לו את הכתובת + המפתח + פורמט ה-JSON (`full_name`, `phone`, `email`) — עבודה של שעה
- **אם האתר בוורדפרס/וויקס:** מחברים את הטופס דרך Make/Zapier (עוד סנריו קטן, כמו מסלול 4) או תוסף webhooks
- ספר לי על מה האתר בנוי — ואתן הוראות מדויקות

✅ **סיום המסלול:** כל שלושת מקורות הלידים (מודעות, אתר, ידני) זורמים למערכת אחת.

## מסלול 6 — Google Calendar (אופציונלי, בהמשך) 👤 + 🤖

**מה כבר יש:** יומן פנימי מלא פר-איש-צוות. הסנכרון לגוגל הוא תוספת נוחות, לא חוסם כלום.

כשתחליט שרוצים: תצטרך ליצור פרויקט ב-Google Cloud Console (חינם), להגדיר OAuth consent screen ולהוציא Client ID/Secret — תהליך של ~30 דק' שאלווה בו מסך-מסך, ואז אני בונה את צד הקוד (תוכנית נפרדת). **המלצה: לעשות רק אחרי שהמערכת חיה ורצה.**
